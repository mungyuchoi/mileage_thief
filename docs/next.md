# 마일캐치 Next: Why 중심 유입/락인 기능 제안

작성일: 2026-05-18

## 1. 핵심 Why

마일캐치의 스플래시 문구는 현재 앱의 Why를 가장 선명하게 말한다.

> 놓치던 여행의 기회를 먼저 캐치하다

이 문구에서 중요한 단어는 `여행`, `기회`, `먼저`다. 마일캐치는 단순한 상품권 시세표, 카드 목록, 커뮤니티, 항공권 특가 앱으로 보이면 안 된다. 사용자가 직접 뒤지고 계산하고 비교하기 전에, 여행으로 이어질 수 있는 기회를 먼저 발견하게 해주는 앱이어야 한다.

현재 코드에는 이 Why를 실현할 재료가 이미 꽤 많이 있다.

- 가이드 탭: 카드, 채팅, 상품권, 특가, 대한항공 마일리지 검색, 구매/판매 등록 진입점
- 마일캐치 레이더: 항공 특가, 상품권, 상품권 특가, 취소표, 뉴스/혜택, 계산기 추천
- 상품권: 정보, 특가, 지도, 시세, 정산, 지점
- 카드: 전체 카드, 추천, 랭킹, 카드사, 이벤트, 카드 상세 피드/혜택/댓글
- 커뮤니티: 게시판, 베스트 글, 라벨링, 좋아요, 댓글, 북마크, 땅콩, 등급, 알림
- 딥링크/공유: Branch 기반 게시글, 카드, 상품권 특가, 커뮤니티, 지점 이동

다음 단계의 방향은 새 기능을 넓게 더 붙이는 것이 아니라, 이미 흩어져 있는 기능을 `오늘 내가 잡아야 할 기회`로 묶는 것이다.

## 2. 제품 포지션

마일캐치가 가져가야 할 포지션은 다음 문장으로 정리할 수 있다.

**마일캐치는 마일리지 여행자를 위한 기회 레이더다.**

상품권 시세는 마일 원가를 낮추는 수단이고, 카드 추천은 마일을 쌓는 수단이며, 커뮤니티는 실전 검증과 제보의 장치다. 항공권/호텔 특가와 좌석 알림은 최종 여행 실행으로 이어지는 트리거다.

따라서 유입 문구와 기능 구조도 `상품권 싸게 사고팔기`보다 `여행 기회를 먼저 잡기`를 중심에 두는 편이 좋다. 상품권과 상테크 기능은 강력한 락인 장치지만, 외부 사용자를 처음 끌어오는 메시지는 더 넓고 직관적인 여행 기회여야 한다.

## 3. 추천 기능 Top 7

### 1위. 오늘의 기회 피드

**정의**  
가이드 탭 첫 화면에 레이더, 관심 라벨 글, 상품권 시세 변동, 카드 이벤트, 항공/호텔 특가를 섞어서 보여주는 개인화 피드.

**왜 필요한가**  
현재 앱은 기능이 많지만 사용자가 각 탭을 직접 돌아다녀야 한다. 유입된 사용자가 첫 화면에서 “아, 오늘 여기서 볼 게 있네”를 느껴야 재방문이 생긴다.

**MVP**
- `UsefulInfoScreen`의 레이더 섹션을 확장해 `오늘의 기회` 블록으로 재구성
- 기존 `RadarService.loadRadarItems()` 결과를 중심으로 표시
- 카드 추천, 상품권 최고 매입가, 상품권 특가, 커뮤니티 인기글을 한 줄 이유와 함께 노출
- 각 카드에는 `왜 이걸 보여주는지`를 짧게 표시

**락인 포인트**
- 사용자가 매일 앱을 열 이유가 생긴다.
- 푸시 없이도 홈 진입 자체가 “오늘 확인할 것”으로 기능한다.

### 2위. 관심 라벨 구독

**정의**  
사용자가 카드, 상품권, 지점, 노선 같은 관심 대상을 팔로우하면 관련 글/시세/알림 후보를 모아주는 기능.

**왜 필요한가**  
라벨링 기능은 이미 커뮤니티 글과 카드/상품권/지점을 연결하고 있다. 다음 단계는 이 연결을 사용자의 관심사로 바꾸는 것이다.

**MVP**
- 저장 경로: `users/{uid}/followedLabels/{labelKey}`
- 라벨 타입: `card`, `giftcard`, `branch`부터 시작
- 카드 상세, 상품권 상세, 지점 상세, 커뮤니티 라벨 칩에 `관심 등록` 추가
- 관심 라벨 기반 피드를 가이드 탭 또는 커뮤니티에 노출

**락인 포인트**
- 사용자가 “내 관심 카드/지점/상품권에 새 글이 올라왔나”를 확인하러 온다.
- 라벨 글 작성자의 글 노출 가치가 커져 커뮤니티 작성 동기가 생긴다.

### 3위. 여행 목표 보드

**정의**  
사용자가 “내년 2월 도쿄 비즈니스 2인” 같은 목표를 만들면 필요한 마일, 좌석/특가 알림, 카드/상품권 전략, 관련 커뮤니티 글을 목표 단위로 묶는 기능.

**왜 필요한가**  
마일캐치의 최종 목적지는 상품권 거래가 아니라 여행이다. 사용자가 여행 목표를 만들면 앱이 월간/장기 루틴으로 바뀐다.

**MVP**
- 저장 경로: `users/{uid}/travelGoals/{goalId}`
- 입력값: 목적지, 기간, 인원, 좌석 등급, 선호 항공사
- 연결값: 레이더 조건, 취소표 알림, 관련 게시판 글, 카드 추천, 상품권 원가 계산
- 목표 상세에 `필요 마일`, `현재 준비 상태`, `다음 액션` 표시

**락인 포인트**
- 목표는 하루짜리 행동이 아니라 몇 주~몇 달짜리 루틴이다.
- 사용자가 알림, 카드, 상품권, 커뮤니티를 하나의 목표 아래 계속 연결하게 된다.

### 4위. 상테크 루트 플래너

**정의**  
보유 카드, 목표 실적, 상품권 시세, 판매 지점, 기존 구매/판매 기록을 바탕으로 이번 달 실행 가능한 상테크 루트를 추천하는 기능.

**왜 필요한가**  
현재 상품권 대시보드와 카드 대시보드는 이미 데이터가 좋다. 하지만 사용자는 “그래서 이번 달에 뭘 해야 하지?”를 알고 싶다.

**MVP**
- 입력값: 월 목표 금액, 보유 카드, 상품권, 선호 지점
- 출력값: 예상 구매액, 예상 판매액, 실적 달성률, 예상 마일, 원/마일
- 기존 `users/{uid}/cards`, `lots`, `sales`, `giftcardRates_current` 활용
- 결과는 “추천”보다 “계산 보조” 톤으로 표시

**락인 포인트**
- 매월 반복해서 들어올 이유가 생긴다.
- 사용자가 내 카드/구매/판매 기록을 쌓을수록 앱을 떠나기 어려워진다.

### 5위. 실거래 검증 배지

**정의**  
상품권 지점/상품권/시세에 `최근 실거래 확인`, `24시간 내 제보`, `가격 변동 주의` 같은 신뢰 신호를 붙이는 기능.

**왜 필요한가**  
상품권 시세에서 중요한 것은 숫자 자체보다 “이 가격을 믿고 움직여도 되는가”다. 커뮤니티 제보가 시세표의 신뢰 계층이 되어야 한다.

**MVP**
- 저장 경로: `branches/{branchId}/tradeVerifications/{verificationId}`
- 필드: 상품권, 가격, 거래일, 메모, 작성자, 선택적 이미지
- 지점/상품권 상세에 최근 제보 수와 마지막 제보 시각 표시
- 커뮤니티 글 작성 시 실거래 라벨을 붙이면 자동 연결

**락인 포인트**
- 사용자가 제보를 남길 이유가 명확해진다.
- 다른 사용자는 제보가 많은 지점을 더 신뢰하게 된다.

### 6위. 공유 가능한 기회 카드

**정의**  
레이더 결과, 카드 추천, 상품권 시세, 상품권 특가, 커뮤니티 라벨 글을 이미지/링크 카드로 공유하는 기능.

**왜 필요한가**  
마일캐치는 정보가 실전적이라 공유 소재가 많다. 하지만 공유가 단순 앱 링크에 머물면 유입력이 약하다. “이 기회” 자체가 공유되어야 한다.

**MVP**
- Branch 딥링크 재사용
- 공유 대상: 레이더 아이템, 카드 상세, 상품권 특가, 지점 시세, 게시글
- 공유 문구: `왜 지금 볼 만한지`, `가격/마일/시세`, `마일캐치에서 보기`
- 이미지 생성은 v1에서는 화면 캡처형 또는 텍스트 카드형으로 단순화

**유입 포인트**
- 커뮤니티/오픈채팅/블로그에서 링크가 자연스럽게 퍼진다.
- 앱 설치 후 바로 해당 카드/특가/지점/글로 진입해 이탈을 줄인다.

### 7위. 땅콩 미션/리텐션 루프

**정의**  
라벨 글쓰기, 실거래 제보, 관심 라벨 등록, 레이더 알림 저장, 목표 보드 업데이트 같은 핵심 행동에 땅콩 보상을 연결하는 기능.

**왜 필요한가**  
땅콩은 이미 커뮤니티와 프로필/이펙트/확장 기능에 연결되어 있다. 다음 단계는 땅콩을 단순 포인트가 아니라 앱의 핵심 데이터 생산 행동에 붙이는 것이다.

**MVP**
- 일일 미션: 관심 라벨 1개 등록, 라벨 글 1개 읽기, 레이더 조건 저장, 실거래 제보
- 주간 미션: 여행 목표 업데이트, 구매/판매 기록 입력, 커뮤니티 검증 글 작성
- 보상은 작게 시작하고, 악용 가능한 행동은 1일 제한을 둔다.

**락인 포인트**
- 재방문과 데이터 생산을 동시에 만든다.
- 사용자가 앱 안에서 작은 진행감을 느낀다.

## 4. 추천 로드맵

### Phase 1. 첫 화면을 Why에 맞춘다

가장 먼저 `오늘의 기회 피드`와 `관심 라벨 구독`을 만든다.

이유:
- 기존 코드 자산을 가장 많이 재사용한다.
- 신규 사용자가 앱의 목적을 빠르게 이해한다.
- 라벨링 기능의 제품적 가치가 즉시 드러난다.

성공 기준:
- 첫 방문 후 3분 내 상세 화면 이동률 증가
- 관심 라벨 등록률
- 레이더 알림 저장률
- 가이드 탭 재방문율

### Phase 2. 외부 유입과 신뢰를 만든다

두 번째는 `공유 가능한 기회 카드`와 `실거래 검증 배지`를 붙인다.

이유:
- 공유는 신규 유입에 직접 연결된다.
- 실거래 검증은 상품권/지점 기능의 신뢰도를 올린다.
- 커뮤니티 글이 단순 게시물이 아니라 데이터가 된다.

성공 기준:
- Branch 링크 클릭/설치/오픈 수
- 공유 시작 수
- 실거래 제보 수
- 제보가 붙은 지점 상세 체류 시간

### Phase 3. 장기 락인을 만든다

세 번째는 `여행 목표 보드`, `상테크 루트 플래너`, `땅콩 미션`을 순차 적용한다.

이유:
- 목표 보드는 장기 재방문을 만든다.
- 루트 플래너는 사용자의 개인 데이터를 앱에 쌓게 만든다.
- 땅콩 미션은 관심 등록, 제보, 기록 입력을 반복 행동으로 만든다.

성공 기준:
- 여행 목표 생성 수
- 목표 보드 재방문율
- 월간 구매/판매 기록 입력 사용자 수
- 7일/30일 리텐션

## 5. 구현 우선순위

1. `오늘의 기회 피드`
2. `관심 라벨 구독`
3. `공유 가능한 기회 카드`
4. `실거래 검증 배지`
5. `여행 목표 보드`
6. `상테크 루트 플래너`
7. `땅콩 미션/리텐션 루프`

이 순서가 좋은 이유는 앞의 기능들이 뒤 기능의 데이터 기반이 되기 때문이다. 관심 라벨이 쌓이면 피드와 알림 품질이 좋아지고, 실거래 제보가 쌓이면 루트 플래너의 신뢰도가 높아진다. 여행 목표가 생기면 카드/상품권/좌석/특가가 사용자의 목적 아래 묶인다.

## 6. 핵심 지표

- 유입: 공유 링크 클릭 수, Branch 딥링크 오픈 수, 설치 후 첫 상세 진입률
- 활성화: 첫 관심 라벨 등록률, 첫 레이더 알림 저장률, 첫 글/댓글/제보 작성률
- 리텐션: 1일/7일/30일 재방문율, 목표 보드 재방문율, 알림 클릭 후 재방문율
- 데이터 생산: 라벨이 붙은 글 비율, 실거래 제보 수, 구매/판매 기록 입력 수
- 락인: 보유 카드 등록 수, 여행 목표 생성 수, 레이더 조건 수, 관심 라벨 수

## 7. 한 줄 결론

마일캐치의 다음 성장은 기능을 더 많이 나열하는 것이 아니라, `놓치던 여행의 기회를 먼저 캐치하다`라는 Why에 맞게 기존 기능들을 개인화된 기회 흐름으로 묶는 데 있다. 첫 화면은 오늘의 기회를 보여주고, 라벨은 관심사를 저장하게 만들고, 알림과 목표 보드는 사용자가 다시 돌아올 이유를 만들어야 한다.
