# 마일캐치 라벨링/피드 기능 블로그 원고

이 문서는 네이버 블로그에 그대로 붙여넣기 위한 원고입니다. 실제 업로드 전에는 `[이미지]` 자리마다 앱 화면 캡처를 넣고, 민감한 닉네임/지점명/금액/작성자 정보가 있으면 모자이크합니다.

## 운영 메모

- 업로드 채널: 네이버 블로그
- 앱 내 재활용 위치: `커뮤니티 > 사용법`
- 대상 독자: 마일캐치에서 커뮤니티, 상품권 지점, 상품권, 카드 정보를 함께 보는 사용자
- 핵심 메시지: 마일캐치의 라벨링은 게시글을 단순한 글이 아니라 `지점`, `상품권`, `카드`에 연결되는 정보 자산으로 만든다.
- 글의 흐름: `WHY → HOW → WHAT`
- 추천 썸네일: 상품권 지점 또는 카드 상세의 `피드` 탭과 커뮤니티 글쓰기 라벨 칩이 함께 보이는 화면

## 제목 후보

1. 마일캐치 커뮤니티가 달라졌습니다: 글이 지점·상품권·카드에 연결되는 라벨링 기능
2. 왜 커뮤니티 글에 라벨이 필요할까? 마일캐치 라벨링과 피드 탭 이야기
3. 상품권 지점, 상품권, 카드 상세에 피드가 생긴 이유
4. 흩어진 상테크 정보를 연결하는 방법: 마일캐치 라벨링 업데이트
5. 커뮤니티 글이 정보가 되는 순간: 마일캐치 라벨링 기능 소개

## 캡처 체크리스트

- [ ] 커뮤니티 게시글 작성 화면: `라벨 추가` 버튼과 선택된 라벨 칩
- [ ] 라벨 추가 화면: 지점/상품권/카드 탭 또는 카드사별 카드 선택 화면
- [ ] 커뮤니티 상세 화면: 제목 아래 라벨 칩 표시
- [ ] 라벨 칩 클릭 후 해당 상품권 지점/상품권/카드 상세로 이동한 화면
- [ ] 상품권 지점 상세: `피드 / 취급 상품권 / 시세·차트 / 리뷰 / 정보` 탭
- [ ] 상품권 지점 피드: 3열 그리드와 `글쓰기` 버튼
- [ ] 상품권 상세: `피드 / 시세 / 정보` 탭
- [ ] 상품권 상세 피드: 라벨링된 커뮤니티 글 그리드
- [ ] 카드 상세: `피드 / 정보 / 혜택 / 댓글` 탭
- [ ] 카드 상세 피드: 카드 라벨 글과 `글쓰기` 버튼
- [ ] 글쓰기 FAB 클릭 후 자동으로 라벨이 선택된 게시글 작성 화면

# 게시글 원고

## 글이 흘러가면 정보가 사라집니다

[이미지 01: 커뮤니티 게시글 목록 또는 커뮤니티 홈 화면]

마일캐치를 만들면서 계속 고민했던 것이 하나 있습니다.

커뮤니티에 좋은 글이 올라와도 시간이 지나면 묻힙니다.

어떤 사용자가 `고고 상품권`에서 롯데상품권을 팔았던 경험을 남겼다고 해볼게요. 그 글은 커뮤니티에서는 분명 의미가 있습니다. 그런데 며칠만 지나도 새 글에 밀리고, 나중에 `고고 상품권` 상세 화면을 보는 사람은 그 글을 다시 찾기 어렵습니다.

카드도 마찬가지입니다.

어떤 카드가 상품권 실적에 도움이 되는지, 어떤 조건에서 마일리지 적립이 괜찮은지, 실제 사용자가 남긴 경험이 커뮤니티에 쌓여도 카드 상세 화면과 연결되지 않으면 정보는 따로 놀게 됩니다.

상품권도 같습니다.

롯데상품권, 신세계상품권, 현대상품권 같은 상품권별로 실제 사용자 글이 쌓여도, 상품권 상세 화면에서 바로 볼 수 없다면 사용자는 다시 검색하고, 다시 게시판을 뒤지고, 다시 판단해야 합니다.

그래서 이번 업데이트의 핵심은 단순합니다.

> 커뮤니티 글을 흘러가는 글로 두지 않고, 필요한 화면에 다시 연결되게 만들자.

이게 이번 라벨링 기능을 만든 이유입니다.

## WHY: 왜 라벨링이 필요했을까?

[이미지 02: 상품권 지점 상세 화면과 커뮤니티 글을 나란히 보여주는 캡처]

마일캐치에는 정보가 많습니다.

- 커뮤니티 글
- 상품권 지점
- 상품권 시세
- 카드 혜택
- 사용자 리뷰
- 이벤트
- 계산기와 차트

문제는 정보가 많아질수록 사용자가 더 편해지는 것이 아니라, 오히려 어디를 봐야 할지 헷갈릴 수 있다는 점입니다.

사용자가 진짜 궁금한 것은 보통 이런 질문입니다.

“이 지점 실제로 괜찮을까?”

“롯데상품권은 지금 어디에서 사고팔면 좋을까?”

“이 카드로 상품권 결제할 때 실제로 도움이 될까?”

“다른 사람들은 이걸 어떻게 쓰고 있을까?”

이 질문에 답하려면 단순한 기본 정보만으로는 부족합니다. 실제 사용자들이 남긴 글, 후기, 질문, 경험이 함께 보여야 합니다.

그래서 커뮤니티 글에 `라벨`을 붙일 수 있게 했습니다.

라벨은 글과 대상을 연결하는 고리입니다.

- 지점 라벨: 특정 상품권 지점과 연결
- 상품권 라벨: 롯데상품권, 신세계상품권 같은 상품권과 연결
- 카드 라벨: 특정 카드 상품과 연결

이제 커뮤니티에 글을 쓸 때 “이 글은 고고 상품권에 관한 글입니다”, “이 글은 롯데상품권에 관한 글입니다”, “이 글은 이 카드에 관한 글입니다”라고 표시할 수 있습니다.

그리고 그 글은 해당 상세 화면의 `피드` 탭에 다시 모입니다.

## HOW: 어떻게 연결했을까?

[이미지 03: 게시글 작성 화면의 라벨 추가 버튼]

커뮤니티 게시글 작성 화면에 `라벨 추가` 기능을 넣었습니다.

글을 쓰다가 관련 있는 대상을 선택할 수 있습니다.

- 상품권 지점
- 상품권
- 카드

라벨은 최대 5개까지 선택할 수 있습니다. 예를 들어 어떤 글이 `고고 상품권`, `롯데상품권`, `현대카드`와 관련 있다면 세 개를 함께 연결할 수 있습니다.

[이미지 04: 라벨 추가 화면 - 지점/상품권/카드 탭]

라벨 추가 화면은 단순 검색만 있는 화면이 아니라, 사용자가 고르기 쉽게 탐색형으로 구성했습니다.

지점 탭에서는 상품권 지점 목록을 볼 수 있습니다.

상품권 탭에서는 롯데상품권, 신세계상품권, 현대상품권처럼 상품권 브랜드를 선택할 수 있습니다.

카드 탭에서는 카드사별로 카드를 보고 선택할 수 있습니다.

즉, 사용자가 정확한 이름을 몰라도 “카드사에서 찾아보기”, “상품권 목록에서 고르기”, “지점 목록에서 고르기”가 가능하도록 했습니다.

[이미지 05: 선택된 라벨 칩이 보이는 게시글 작성 화면]

라벨을 선택하면 게시글 작성 화면에 칩 형태로 표시됩니다.

이 칩은 단순한 태그가 아닙니다. 글이 저장될 때 마일캐치는 이 라벨 정보를 함께 저장하고, 해당 지점·상품권·카드 상세 화면에서도 이 글을 다시 찾을 수 있게 연결합니다.

## WHAT 1: 커뮤니티 상세에서 라벨을 바로 누를 수 있습니다

[이미지 06: 커뮤니티 상세 화면 제목 아래 라벨 칩]

게시글 상세 화면에서는 제목과 작성 시간 아래에 라벨 칩이 보입니다.

예를 들어 글에 `고고 상품권`, `롯데상품권`, `현대카드` 라벨이 붙어 있다면, 상세 화면에서 해당 라벨이 칩으로 표시됩니다.

사용자는 칩을 누르기만 하면 바로 연결된 화면으로 이동할 수 있습니다.

- `고고 상품권` 라벨 클릭 → 상품권 지점 상세
- `롯데상품권` 라벨 클릭 → 상품권 상세
- `현대카드` 라벨 클릭 → 카드 상세

[이미지 07: 라벨 칩 클릭 후 상세 화면으로 이동한 화면]

이 흐름이 중요한 이유는 사용자가 정보를 읽다가 바로 맥락으로 이동할 수 있기 때문입니다.

글을 읽고 “이 지점이 어디지?” 하고 다시 검색할 필요가 없습니다.

글을 읽고 “이 카드가 어떤 카드지?” 하고 카드 목록을 다시 뒤질 필요도 없습니다.

라벨을 누르면 바로 연결됩니다.

## WHAT 2: 상품권 지점 상세가 피드 중심으로 바뀌었습니다

[이미지 08: 상품권 지점 상세 상단 화면]

상품권 지점 상세 화면도 구조를 바꿨습니다.

기존에는 지점 정보, 시세, 리뷰가 한 화면에 섞여 있었다면 이제는 탭으로 나눠 더 보기 쉽게 구성했습니다.

대표 흐름은 다음과 같습니다.

- `피드`: 이 지점을 라벨링한 커뮤니티 글
- `취급 상품권`: 지점이 취급하는 상품권
- `시세·차트`: 상품권별 시세와 변화
- `리뷰`: 지점 리뷰
- `정보`: 주소, 전화, URL, 안내사항 등

[이미지 09: 상품권 지점 피드 탭 3열 그리드]

가장 먼저 보이는 탭은 `피드`입니다.

이 지점을 라벨링한 글이 있으면 인스타 피드처럼 3열 그리드로 보여줍니다.

이미지가 있는 글은 이미지로 보이고, 이미지가 없는 글은 제목과 본문 일부가 텍스트 카드로 표시됩니다.

이제 지점 상세 화면은 단순한 정적 정보 화면이 아니라, 해당 지점에 대한 사용자 글이 쌓이는 공간이 됩니다.

[이미지 10: 상품권 지점 피드의 글쓰기 버튼]

피드 탭에는 `글쓰기` 버튼도 있습니다.

이 버튼을 누르면 바로 `적립/카드 혜택` 게시판으로 글쓰기 화면이 열리고, 현재 보고 있던 지점 라벨이 자동으로 선택됩니다.

사용자는 게시판을 다시 고르거나 라벨을 다시 찾을 필요 없이 바로 글을 쓰면 됩니다.

## WHAT 3: 상품권 상세도 피드와 시세를 분리했습니다

[이미지 11: 롯데상품권 상세 화면]

상품권 자체의 상세 화면도 새로 정리했습니다.

예를 들어 `롯데상품권`을 눌렀을 때 이제는 단순히 지점별 시세 리스트만 보는 것이 아니라, 상품권 하나를 기준으로 정보를 볼 수 있습니다.

탭 구성은 다음과 같습니다.

- `피드`: 롯데상품권을 라벨링한 커뮤니티 글
- `시세`: 지점별 롯데상품권 매입/판매 시세
- `정보`: 최고/최저 시세, 기준 지점, 업데이트 시간, 상품권 ID 등

[이미지 12: 상품권 상세 피드 탭]

상품권 피드도 3열 그리드로 표시됩니다.

이제 어떤 사용자가 `롯데상품권` 라벨을 붙여 글을 쓰면, 그 글은 커뮤니티에도 남고 롯데상품권 상세 화면에도 모입니다.

이렇게 되면 상품권별로 사용자 경험이 축적됩니다.

롯데상품권을 보는 사용자는 롯데상품권과 관련된 글을 바로 볼 수 있고, 신세계상품권을 보는 사용자는 신세계상품권과 관련된 글을 바로 볼 수 있습니다.

[이미지 13: 상품권 상세 피드 글쓰기 버튼]

상품권 상세의 피드 탭에서도 `글쓰기` 버튼을 제공합니다.

버튼을 누르면 `적립/카드 혜택` 게시판으로 바로 이동하고, 현재 상품권 라벨이 자동 선택됩니다.

예를 들어 롯데상품권 상세에서 글쓰기 버튼을 누르면, 글쓰기 화면에 `롯데상품권` 라벨이 이미 붙어 있는 상태로 시작합니다.

## WHAT 4: 카드 상세도 피드, 정보, 혜택, 댓글로 나눴습니다

[이미지 14: 카드 상세 화면의 탭 구조]

카드 상세 화면도 같은 방향으로 정리했습니다.

기존에는 기본 정보, 주요 혜택, 댓글이 한 화면에 길게 이어졌습니다. 이제는 탭을 나눠 필요한 정보를 더 빠르게 볼 수 있게 했습니다.

탭 구성은 다음과 같습니다.

- `피드`: 이 카드를 라벨링한 커뮤니티 글
- `정보`: 카드사, 카드 유형, 이벤트, 상세 정보, 카드사 바로가기
- `혜택`: 주요 혜택, 마일리지/여행, 상테크 계산, 제외/유의 항목
- `댓글`: 기존 카드 전용 댓글

[이미지 15: 카드 상세 피드 탭]

카드 상세의 첫 탭도 `피드`입니다.

이 카드를 라벨링한 커뮤니티 글이 있으면 3열 그리드로 보여줍니다.

카드는 특히 사용자 경험이 중요합니다.

공식 혜택에는 적혀 있지만 실제로는 조건이 까다로운 카드도 있고, 상품권 결제나 마일리지 적립처럼 사용자가 직접 확인해야 하는 부분도 많습니다.

그래서 카드 상세 화면에서도 커뮤니티 글이 함께 보여야 합니다.

[이미지 16: 카드 상세 피드 글쓰기 버튼]

카드 상세 피드에서도 `글쓰기` 버튼을 누르면 바로 게시글 작성 화면으로 이동합니다.

이때 게시판은 `적립/카드 혜택`으로 고정되고, 현재 카드 라벨이 자동으로 선택됩니다.

즉, 사용자는 “이 카드에 대해 글을 써야겠다”는 순간 바로 작성할 수 있습니다.

## 라벨링이 바꾸는 것은 화면이 아니라 흐름입니다

[이미지 17: 커뮤니티 글 → 라벨 → 상세 피드로 이어지는 흐름을 보여주는 캡처 조합]

이번 업데이트를 단순히 “태그 기능이 생겼다”고 보면 조금 아쉽습니다.

핵심은 정보의 흐름이 바뀌었다는 점입니다.

이전에는 커뮤니티 글이 커뮤니티 안에만 있었습니다.

이제는 글이 필요한 곳으로 다시 돌아갑니다.

사용자가 지점에 대해 글을 쓰면 그 글은 지점 상세에 모입니다.

사용자가 상품권에 대해 글을 쓰면 그 글은 상품권 상세에 모입니다.

사용자가 카드에 대해 글을 쓰면 그 글은 카드 상세에 모입니다.

정보가 흩어지는 것이 아니라 연결됩니다.

## 사용자는 이렇게 쓰면 됩니다

[이미지 18: 게시글 작성 화면 전체]

커뮤니티에서 글을 쓸 때는 이렇게 하면 됩니다.

1. 커뮤니티에서 글쓰기 버튼을 누릅니다.
2. 제목과 내용을 작성합니다.
3. `라벨 추가`를 누릅니다.
4. 관련 지점, 상품권, 카드를 선택합니다.
5. 선택된 라벨을 확인하고 글을 등록합니다.

또는 상세 화면에서 바로 시작할 수도 있습니다.

1. 상품권 지점, 상품권, 카드 상세 화면으로 들어갑니다.
2. 첫 번째 탭인 `피드`를 봅니다.
3. `글쓰기` 버튼을 누릅니다.
4. 라벨이 자동 선택된 상태에서 글을 작성합니다.

두 방식 모두 결과는 같습니다.

글은 커뮤니티에 올라가고, 동시에 관련 상세 화면의 피드에도 연결됩니다.

## 예시로 보면 더 쉽습니다

[이미지 19: 고고 상품권 지점 피드에 글이 쌓인 화면]

예를 들어 `고고 상품권`에서 상품권을 팔아본 경험을 남기고 싶다면, 고고 상품권 지점 상세의 피드에서 `글쓰기`를 누르면 됩니다.

그러면 글쓰기 화면에 `고고 상품권` 라벨이 자동으로 붙습니다.

글을 등록하면 그 글은 커뮤니티에도 올라가고, 고고 상품권 상세 피드에도 표시됩니다.

[이미지 20: 롯데상품권 피드에 글이 쌓인 화면]

롯데상품권 관련 글이라면 `롯데상품권` 라벨을 붙이면 됩니다.

그러면 롯데상품권 상세 화면의 피드에서 그 글을 다시 볼 수 있습니다.

[이미지 21: 카드 상세 피드에 글이 쌓인 화면]

카드도 마찬가지입니다.

특정 카드의 상품권 실적, 마일리지 적립, 실제 사용 경험을 남기고 싶다면 카드 상세 피드에서 바로 글을 쓰면 됩니다.

그 글은 해당 카드의 피드에 쌓입니다.

## 마무리: 마일캐치는 정보를 연결하는 앱이 되려고 합니다

[이미지 22: 지점 피드, 상품권 피드, 카드 피드를 한 화면에 모은 콜라주]

마일캐치가 하고 싶은 일은 단순히 화면을 많이 만드는 것이 아닙니다.

사용자가 판단할 때 필요한 정보를 그 자리에서 볼 수 있게 만드는 것입니다.

상품권 지점을 볼 때는 그 지점에 대한 글이 보여야 합니다.

상품권을 볼 때는 그 상품권에 대한 글이 보여야 합니다.

카드를 볼 때는 그 카드에 대한 글이 보여야 합니다.

그래야 사용자는 커뮤니티를 따로 뒤지지 않아도, 지금 보고 있는 대상에 대한 실제 경험을 바로 확인할 수 있습니다.

이번 라벨링 기능은 그 시작입니다.

> 정보는 흩어지면 사라지고, 연결되면 자산이 됩니다.

마일캐치는 커뮤니티 글 하나하나가 시간이 지나도 다시 필요한 곳에서 발견될 수 있도록 계속 다듬어가겠습니다.
