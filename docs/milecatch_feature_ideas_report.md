# 마일캐치 신규 기능 아이디어 5선 보고서

작성일: 2026-05-16

## 1. Executive Summary

마일캐치의 스플래시 문구는 다음 한 문장으로 제품의 Why를 설명한다.

> 놓치던 여행의 기회를 먼저 캐치하다

이 문장은 단순히 항공권 좌석을 빨리 찾는다는 뜻에 그치지 않는다. 사용자가 놓치기 쉬운 기회는 좌석, 카드 혜택, 상품권 시세, 지점 실거래 정보, 커뮤니티 제보, 호텔/항공 특가까지 여러 곳에 흩어져 있다. 마일캐치가 앞으로 강화해야 할 방향은 이 조각들을 한 화면에 모아 “지금 내가 잡아야 할 기회”로 번역해주는 것이다.

오늘 추가한 라벨링 기반 연결은 이 방향의 핵심 기반이다. 커뮤니티 글이 카드, 상품권, 지점 같은 실체와 연결되면, 글은 단순한 게시물이 아니라 특정 기회에 붙는 해설, 검증, 후기, 알림 데이터가 된다. 따라서 다음 기능은 조회 기능을 늘리는 것보다 라벨링을 중심으로 정보, 판단, 실행, 공유가 이어지도록 설계해야 한다.

본 보고서는 1-2개월 안에 MVP로 만들 수 있고, 커뮤니티 리텐션과 데이터 네트워크 효과를 키울 가능성이 높은 기능 5개를 선별했다.

## 2. Selection Criteria

아이디어 선별 기준은 다음 4가지다.

- **Why 적합성**: 사용자가 놓칠 수 있는 여행/마일리지/상테크 기회를 먼저 발견하게 하는가.
- **커뮤니티 리텐션**: 사용자가 다시 들어와 확인하고, 글을 남기고, 제보할 이유를 만드는가.
- **기존 자산 활용도**: 좌석, 상품권, 지점, 카드, 라벨, 레이더, 커뮤니티 데이터를 재사용하는가.
- **MVP 현실성**: 1-2개월 내 Flutter/Firebase 구조 위에서 출시 가능한가.

외부 시장 신호도 함께 고려했다. 2026년 카드사는 고혜택 상품을 줄이는 흐름이 강하고, 상품권 시세/거래 서비스는 이미 많다. 이 환경에서 마일캐치가 단순 시세표나 카드 목록으로 경쟁하면 방어력이 약하다. 차별점은 “실행 가능한 개인화 기회”와 “커뮤니티 기반 검증”이다.

참고 신호:

- 카드 고혜택 상품 축소: Seoul Economic Daily, “Korean Card Issuers Discontinue 500 Products Annually Amid Profit Squeeze”
- 상품권 P2P/시세 경쟁: PIN2P, 지지앤핀 등 상품권 거래/시세 서비스
- 해외 churning 커뮤니티 흐름: 카드/마일리지 최적화는 커뮤니티 규범과 실전 제보를 중심으로 발전

## 3. Top 5 Feature Ideas

### 1위. 기회 지도: 라벨 기반 개인화 피드

**한 줄 정의**  
사용자가 관심 있는 카드, 상품권, 지점, 노선, 특가 라벨을 구독하면 관련 글, 시세, 알림, 계산기, 최근 변동을 한 화면에 모아주는 개인화 피드다.

**왜 마일캐치다운가**  
마일캐치의 핵심은 “기회를 먼저 캐치”하는 것이다. 그런데 지금 기회는 여러 화면에 나뉘어 있다. 사용자는 카드 화면에서 카드 정보를 보고, 상품권 화면에서 시세를 보고, 커뮤니티에서 후기를 찾아야 한다. 기회 지도는 이 흐름을 뒤집는다. 사용자가 관심 대상을 정하면, 마일캐치가 관련 기회를 모아준다.

**사용자 시나리오**

- 사용자가 `신세계상품권`, `롯데상품권`, `현대카드`, `명동 지점`, `도쿄 노선`을 관심 라벨로 등록한다.
- 홈 또는 별도 탭에서 해당 라벨에 연결된 최근 커뮤니티 글, 시세 변화, 특가, 알림 후보가 보인다.
- “신세계상품권 매입가 상승”, “현대카드 관련 새 글”, “명동 지점 실거래 후기”, “도쿄 좌석 제보”가 하나의 피드에 섞여 나온다.
- 사용자는 피드에서 바로 글 작성, 알림 등록, 계산기 이동, 지점 상세 이동을 한다.

**MVP 범위**

- 관심 라벨 저장: `users/{uid}/followedLabels/{labelKey}` 또는 유사 구조.
- 개인화 피드 화면: 라벨 인덱스(`labeledPosts`)와 기존 시세/카드/지점 데이터를 조합해 최신순 카드 표시.
- 관심 라벨 추가 진입점: 카드 상세, 상품권 상세, 지점 상세, 커뮤니티 라벨 칩.
- 초기 추천: 최근 본 카드/상품권/지점을 관심 후보로 제안.

**기대효과**

- 커뮤니티 글의 소비 경로가 게시판 중심에서 관심 대상 중심으로 넓어진다.
- 사용자가 앱을 다시 여는 이유가 생긴다. “내 관심 라벨에 새 기회가 생겼는지” 확인하게 된다.
- 라벨링의 가치가 즉시 체감된다. 글 작성자가 라벨을 붙일수록 더 많은 독자에게 도달한다.

**리스크**

- 초기에는 라벨이 충분히 붙은 글이 적을 수 있다.
- 피드 품질이 낮으면 단순 최신글 목록처럼 보일 수 있다.

**방어 논리**

- 첫 버전은 라벨 기반 커뮤니티 글만으로 시작하지 말고, 시세 변동/카드 정보/지점 정보까지 같이 섞어 빈 화면을 줄인다.
- 라벨 없는 글도 게시판 기준 추천으로 보완하되, 라벨 글을 우선 노출한다.

**우선순위 판단**  
가장 먼저 만들 기능으로 추천한다. 오늘 추가한 라벨 연결의 가치를 가장 빨리 제품 경험으로 바꿀 수 있고, 이후 2-5위 기능의 기반이 된다.

### 2위. 상테크 루트 플래너

**한 줄 정의**  
사용자의 보유 카드, 목표 실적, 관심 상품권, 선호 지점, 현재 시세를 바탕으로 이번 달 실행 가능한 상테크 루트를 추천한다.

**왜 마일캐치다운가**  
상품권 시세표는 “어디가 비싼가”를 알려준다. 카드 카탈로그는 “어떤 카드가 좋은가”를 알려준다. 하지만 사용자가 실제로 원하는 것은 “그래서 이번 달 나는 어떻게 해야 하는가”다. 루트 플래너는 마일캐치가 정보를 넘어 실행 판단을 돕는 기능이다.

**사용자 시나리오**

- 사용자가 보유 카드와 월 실적 목표를 입력한다.
- 앱은 관심 상품권과 지점 시세를 기준으로 구매/판매 후보를 계산한다.
- “신세계 10만권 20장, A카드 실적 반영, B지점 매입가 기준 예상 손익”처럼 실행 단위로 보여준다.
- 관련 커뮤니티 글과 지점 후기를 함께 보여줘 사용자가 위험 요소를 확인한다.

**MVP 범위**

- 입력값: 월 목표 금액, 보유 카드 선택, 상품권 선택, 선호 지점 선택.
- 계산값: 예상 구매액, 예상 판매액, 수수료/할인율, 단순 손익, 실적 달성률.
- 연결 데이터: `giftcards`, `branches/{branchId}/giftcardRates_current`, 카드 카탈로그, 라벨 커뮤니티 글.
- 저장: 추천 루트는 로컬 또는 사용자 하위 컬렉션에 저장해 “이번 달 계획”으로 재진입 가능하게 한다.

**기대효과**

- 상품권/카드/지점 기능이 하나의 목적형 워크플로로 연결된다.
- 사용자가 매월 반복해서 들어올 이유가 생긴다.
- 커뮤니티 글은 루트의 근거/주의사항으로 붙어 실전성이 높아진다.

**리스크**

- 카드별 실적 인정/제외 규칙이 복잡하고 자주 바뀐다.
- 손익 추천이 틀리면 신뢰를 잃을 수 있다.

**방어 논리**

- MVP에서는 “확정 추천”이 아니라 “계산 보조”로 표현한다.
- 카드 규칙은 운영자 입력/커뮤니티 제보 기반으로 시작하고, 최종 판단 책임은 사용자가 확인하도록 유의 문구를 둔다.
- 수익을 과장하지 않고 실적 달성/비용 비교 중심으로 표현한다.

### 3위. 실거래 검증 배지

**한 줄 정의**  
지점, 상품권, 시세, 커뮤니티 글에 “실거래 확인”, “최근 제보”, “가격 변동 주의” 같은 신뢰 배지를 붙이는 기능이다.

**왜 마일캐치다운가**  
상품권 시세 경쟁에서 중요한 것은 숫자 자체가 아니라 “이 숫자를 믿고 움직여도 되는가”다. 사용자는 100원 차이보다 헛걸음, 지점별 조건, 갑작스러운 매입가 변경을 더 두려워한다. 실거래 검증 배지는 커뮤니티를 가격표의 신뢰 계층으로 만든다.

**사용자 시나리오**

- 사용자가 지점에서 상품권을 판매한 뒤 “실거래 확인”을 남긴다.
- 다른 사용자는 지점/상품권 화면에서 “오늘 3건 확인”, “최근 24시간 제보 있음”을 본다.
- 시세와 커뮤니티 글이 연결되어 어떤 조건에서 거래됐는지 확인할 수 있다.
- 오래된 제보는 자동으로 신뢰도가 낮아진다.

**MVP 범위**

- 지점/상품권 상세에 “실거래 제보” 버튼 추가.
- 제보 필드: 상품권, 지점, 매입/판매, 가격, 거래일, 메모, 선택적 사진.
- 표시: 최근 제보 수, 마지막 제보 시각, 대표 가격, 관련 글 목록.
- 커뮤니티 글 작성 시 실거래 라벨을 붙이면 해당 지점/상품권에 자동 인덱싱.

**기대효과**

- 커뮤니티 참여 이유가 명확해진다. 제보가 다른 사용자에게 바로 도움이 된다.
- 시세 데이터의 신뢰도를 높인다.
- 지점별 운영 현실, 조건, 예외 사항이 자연스럽게 축적된다.

**리스크**

- 허위 제보, 광고성 제보, 경쟁 지점 공격 가능성이 있다.
- 개인정보나 영수증 이미지 처리 이슈가 생길 수 있다.

**방어 논리**

- MVP에서는 사진 필수 대신 선택으로 두고, 신고/숨김 기능을 붙인다.
- 사용자 등급, 작성 이력, 반복 제보 정확도를 기준으로 배지 가중치를 다르게 줄 수 있다.
- 가격 확정 표현은 피하고 “사용자 제보”임을 명확히 표시한다.

### 4위. 기회 레이더 고도화

**한 줄 정의**  
기존 알림을 조건 저장형에서 관심 라벨과 비정상 기회 탐지형으로 확장한다.

**왜 마일캐치다운가**  
사용자가 앱을 열어야만 찾을 수 있는 정보는 “먼저 캐치”가 아니다. 좋은 기회는 발생 시점에 알려줘야 한다. 현재 레이더/상품권 특가 알림 구조가 있으므로, 이를 라벨과 개인화 기준으로 넓히면 비교적 빠르게 확장할 수 있다.

**사용자 시나리오**

- 사용자가 `신세계상품권`, `현대카드`, `도쿄`, `비즈니스석`을 관심 라벨로 둔다.
- 평소보다 좋은 상품권 시세, 관련 카드 혜택 변경 글, 급상승 커뮤니티 글, 좌석/특가가 생기면 알림을 받는다.
- 알림을 누르면 단순 상세가 아니라 “왜 이 알림이 왔는지”와 연결 글/시세/계산이 보인다.

**MVP 범위**

- 기존 레이더 알림 화면에 라벨 기반 알림 구독 추가.
- 알림 후보: 라벨 커뮤니티 글, 상품권 특가, 시세 변동, 카드 관련 새 글.
- 알림 기준: 새 글 발생, 특정 라벨의 일정 시간 내 반응 증가, 관심 상품권 가격 조건 만족.
- 알림 히스토리에 라벨과 근거 데이터를 표시.

**기대효과**

- 사용자가 앱을 열지 않아도 재방문 트리거가 생긴다.
- 라벨 구독 기능의 가치를 강화한다.
- 커뮤니티 글 작성자가 더 많은 노출을 얻는다.

**리스크**

- 알림이 많아지면 피로도가 생긴다.
- “좋은 기회” 판정 기준이 초기에 거칠 수 있다.

**방어 논리**

- 기본값은 보수적으로 두고, 사용자가 라벨별 알림 강도를 조절하게 한다.
- 초반에는 ML이 아니라 명확한 규칙 기반으로 시작한다.
- 알림마다 “알림 사유”를 표시해 사용자가 납득할 수 있게 한다.

### 5위. 여행 목표 보드

**한 줄 정의**  
사용자가 여행 목표를 만들면 필요한 마일, 좌석 알림, 카드/상품권 전략, 관련 커뮤니티 글을 목표 단위로 묶어주는 기능이다.

**왜 마일캐치다운가**  
마일캐치의 최종 사용자는 상품권을 팔고 카드 실적을 채우는 사람이 아니라, 결국 여행 기회를 잡고 싶은 사람이다. 여행 목표 보드는 기능 중심 화면을 사용자 목적 중심으로 재구성한다.

**사용자 시나리오**

- 사용자가 “내년 2월 도쿄 비즈니스 2인” 목표를 만든다.
- 앱은 필요한 마일리지, 좌석 알림 조건, 관련 노선 글, 추천 카드/상품권 전략을 묶어 보여준다.
- 목표 진행률이 표시되고, 새 좌석/특가/관련 글이 생기면 목표 보드에 업데이트된다.

**MVP 범위**

- 목표 생성: 목적지, 기간, 인원, 좌석 등급, 항공사 선택.
- 목표 상세: 필요한 마일, 저장된 알림, 관련 커뮤니티 글, 관련 특가/호텔 링크.
- 기존 좌석 알림/레이더와 연결.
- 목표별 메모와 체크리스트 제공.

**기대효과**

- 마일캐치가 “여행을 위한 운영체제”처럼 느껴진다.
- 좌석/카드/상품권/커뮤니티 기능이 사용자 목표 아래 정리된다.
- 장기 리텐션에 강하다. 여행 목표는 하루 만에 끝나는 작업이 아니다.

**리스크**

- 항공사별 마일 차감표, 성수기, 유류할증료 등 계산 난이도가 높다.
- 초기 데이터가 부족하면 단순 메모장처럼 보일 수 있다.

**방어 논리**

- MVP에서는 정교한 자동 계산보다 목표 저장, 알림 연결, 관련 글 추천에 집중한다.
- 마일 계산은 현재 보유 데이터가 충분한 항공사/노선부터 제한적으로 시작한다.

## 4. Recommended Roadmap

### Phase 1: 라벨 기반 리텐션 기반 만들기

가장 먼저 **기회 지도**를 만든다.

이유:

- 오늘 추가한 라벨링 기능의 제품적 의미를 즉시 사용자 경험으로 바꾼다.
- 이후 실거래 검증, 레이더 고도화, 루트 플래너가 모두 라벨 피드 위에 붙을 수 있다.
- 개발 범위가 비교적 작다. 새 데이터 모델은 관심 라벨 저장 정도이며, 핵심은 기존 라벨 인덱스를 잘 보여주는 것이다.

성공 지표:

- 관심 라벨 등록 사용자 수
- 관심 라벨 피드 재방문율
- 라벨이 붙은 게시글 비율
- 라벨 피드에서 게시글/지점/카드/상품권 상세로 이동한 횟수

### Phase 2: 실행 보조 기능 붙이기

두 번째로 **상테크 루트 플래너** 또는 **실거래 검증 배지**를 붙인다.

추천 순서는 운영 상황에 따라 다르다.

- 데이터 신뢰를 먼저 높이고 싶다면 실거래 검증 배지.
- 월간 반복 사용과 체류 시간을 먼저 늘리고 싶다면 상테크 루트 플래너.

### Phase 3: 알림과 목표로 장기 리텐션 만들기

기회 레이더 고도화와 여행 목표 보드는 라벨 피드가 어느 정도 작동한 뒤 붙이는 것이 좋다. 두 기능 모두 개인화 데이터가 쌓일수록 품질이 좋아진다.

## 5. Final Recommendation

가장 먼저 만들 기능은 **기회 지도: 라벨 기반 개인화 피드**다.

이 기능은 마일캐치의 Why와 오늘 추가한 라벨 연결 기능을 가장 직접적으로 이어준다. 카드, 상품권, 지점, 커뮤니티, 알림, 특가 기능을 한 번에 새로 만드는 것이 아니라, 이미 있는 기능들을 사용자 관심사 중심으로 다시 묶는다. 따라서 구현 부담은 상대적으로 낮고, 제품 방향성은 가장 크게 바꿀 수 있다.

마일캐치가 앞으로 경쟁해야 할 대상은 단순 시세표 앱이나 항공권 검색 앱이 아니다. 사용자가 “내가 놓치면 안 되는 기회”를 가장 먼저 확인하는 앱이 되어야 한다. 기회 지도는 그 방향으로 가는 첫 화면이다.
