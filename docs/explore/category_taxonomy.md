# 마일고사 카테고리 분류 설계

작성일: 2026-05-30

## 1. 결론

마일고사 RPG의 첫 메뉴는 너무 잘게 쪼개지 않고 4개 대주제로 시작한다.

```text
호텔
항공
카드/포인트
상품권
```

구조는 아래처럼 잡는다.

```text
대주제 = 분야 Domain
중주제 = 브랜드/프로그램/과정 Course
소주제 = 실제 레슨 Lesson
```

예시:

```text
호텔
-> Marriott
-> 등급 마을
-> 조식/라운지
-> 포인트 가치
-> 숙박권
-> 예약 실전
```

이 구조는 Flutter 화면과 Firestore 모델에도 그대로 대응된다.

```text
explore/main/domains/{domainId}
explore/main/courses/{courseId}
explore/main/courses/{courseId}/sections/{sectionId}
explore/main/courses/{courseId}/lessons/{lessonId}
```

## 2. 대주제 1: 호텔

호텔은 가장 RPG화하기 좋은 분야다. 브랜드별로 "왕국", "성", "등급 마을", "포인트 광산" 같은 표현을 붙이기 쉽고, 사용자가 실제 여행에서 체감하는 이득도 크다.

| 대주제 | 중주제 | 소주제 후보 |
| --- | --- | --- |
| 호텔 | Marriott | 등급, 플래티넘, 조식/라운지, 포인트 숙박, 숙박권, 5박 혜택, 예약 실전 |
| 호텔 | Hilton | 등급, 조식/크레딧, 포인트 숙박, 5박 혜택, 프로모션 |
| 호텔 | Hyatt | 글로벌리스트, 스위트 업그레이드, 포인트 차감, 무료숙박권 |
| 호텔 | IHG | 다이아몬드, 앰배서더, 숙박권, 포인트 가치 |
| 호텔 | Accor | ALL 포인트, 등급, 현금처럼 쓰는 구조 |
| 호텔 | 호텔 공통 | OTA vs 공식홈, 포인트 가치 계산, 취소 규정, 세금/리조트피 |

초기 MVP는 `Marriott`만 열고 나머지는 `준비중`으로 둔다.

추천 과정:

```text
호텔
-> Marriott
-> Hilton 준비중
-> Hyatt 준비중
-> IHG 준비중
-> Accor 준비중
```

## 3. 대주제 2: 항공

항공은 관심도는 높지만 난이도가 높다. 그래서 첫 버전은 "마일 발권 고급 전략"보다 "대한항공 기초"처럼 초보자 진입 과정으로 시작하는 편이 좋다.

| 대주제 | 중주제 | 소주제 후보 |
| --- | --- | --- |
| 항공 | 대한항공 | 보너스 항공권, 가족합산, 좌석 등급, 성수기, 유류할증료, 마일 가치 |
| 항공 | 아시아나 | 마일 사용, 스타얼라이언스, 가족합산, 전환/소멸 |
| 항공 | ANA | 왕복 발권, 스타얼라이언스, 유류할증료, 마일 가치 |
| 항공 | JAL | 원월드, 일본/미주 발권, 파트너 항공 |
| 항공 | 항공 공통 | 이코노미/비즈니스 가치 비교, 마일 좌석 찾기, 취소/변경 수수료 |

추천 과정:

```text
항공
-> 대한항공 기초
-> 아시아나 준비중
-> ANA 준비중
-> JAL 준비중
-> 항공 마일 공통
```

## 4. 대주제 3: 카드/포인트

카드/포인트는 사용자의 소비 판단과 바로 연결된다. 호텔, 항공, 상품권을 이해하기 위한 기반이기도 하다.

| 대주제 | 중주제 | 소주제 후보 |
| --- | --- | --- |
| 카드/포인트 | 카드 기초 | 전월실적, 적립률, 할인 vs 적립, 연회비 판단 |
| 카드/포인트 | 마일리지 카드 | 대한항공 적립, 아시아나 적립, 전환 비율, 실적 제외 |
| 카드/포인트 | 카드사 포인트 | 삼성, 현대, 신한, KB, 롯데 포인트 |
| 카드/포인트 | 전환 전략 | 카드 포인트에서 항공/호텔/상품권으로 전환 |
| 카드/포인트 | 발급/혜택 실전 | 신규 발급, 이벤트, 가족카드, 유지/해지 판단 |

추천 과정:

```text
카드/포인트
-> 카드 기초
-> 마일리지 카드
-> 포인트 전환
-> 발급 이벤트 실전
```

카드/포인트는 `카드 기초`를 먼저 열면 초보자 흡수가 좋다.

## 5. 대주제 4: 상품권

상품권은 계산형 학습과 가장 잘 맞는다. 듀오링고 수학처럼 `빈칸 채우기`, `수치선`, `계산 문제`, `손익분기 판단`으로 만들기 좋다.

| 대주제 | 중주제 | 소주제 후보 |
| --- | --- | --- |
| 상품권 | 상품권 기초 | 할인율, 실구매가, 현금화, 매입가 |
| 상품권 | 상테크 계산 | 수익률, 카드 실적, 수수료, 손익분기 |
| 상품권 | 브랜드별 상품권 | 신세계, 롯데, 현대, 컬처, 해피머니 등 |
| 상품권 | 위험 관리 | 한도, 정책 변경, 환불, 매입가 변동 |
| 상품권 | 실전 문제 | "이 조건이면 사도 되나?" 계산 문제 |

추천 과정:

```text
상품권
-> 상품권 기초
-> 상테크 계산
-> 실전 손익 계산
-> 브랜드별 상품권
```

## 6. 첫 화면 추천

Flutter 첫 화면은 대주제 4개를 상단 탭 또는 아이콘 카드로 보여준다.

```text
[호텔] [항공] [카드/포인트] [상품권]
```

각 메뉴 안에는 과정 카드를 배치한다.

```text
호텔
- Marriott 진행중
- Hilton 준비중
- Hyatt 준비중
- IHG 준비중

항공
- 대한항공 기초
- 아시아나 준비중
- ANA 준비중

카드/포인트
- 카드 기초
- 마일리지 카드
- 포인트 전환

상품권
- 상품권 기초
- 상테크 계산
- 실전 손익 계산
```

## 7. MVP 우선순위

추천 우선순위는 아래와 같다.

| 순서 | 과정 | 이유 |
| ---: | --- | --- |
| 1 | `호텔 > Marriott` | RPG화하기 좋고, 호텔 등급/포인트/숙박권을 한 번에 다룰 수 있다. |
| 2 | `상품권 > 상품권 기초/계산` | 문제화하기 좋고, 계산형 인터랙션과 잘 맞는다. |
| 3 | `항공 > 대한항공 기초` | 유저 관심이 높고, 마일고사 정체성과 잘 맞는다. |
| 4 | `카드/포인트 > 카드 기초` | 모든 분야의 기반이며 소비 판단과 바로 연결된다. |

## 9. Flutter 메뉴 구현 기준

추천 위젯:

- `DomainTabBar`: 호텔/항공/카드·포인트/상품권 전환
- `CourseCardGrid`: 과정 카드 목록
- `CourseCard`: 진행중/시작하기/준비중 상태 표시
- `ContinueLessonCard`: 마지막으로 하던 레슨 이어하기
- `ComingSoonCourseCard`: 잠긴 과정 표시

과정 카드는 최소 아래 정보를 가진다.

```json
{
  "courseId": "marriott_basics",
  "domainId": "hotel",
  "title": "Marriott",
  "subtitle": "포인트와 숙박권으로 호텔을 정복하는 길",
  "status": "published",
  "progressLabel": "진행률 2/18",
  "currentLessonTitle": "등급 마을",
  "iconAsset": "asset/icon/icon_marriott.svg"
}
```

## 10. 운영 원칙

- 첫 화면의 대주제는 4개 이상 늘리지 않는다.
- 중주제는 브랜드와 목적이 분명한 과정만 노출한다.
- 준비 중 과정도 보여주되, 너무 많이 보여줘서 산만해지지 않게 한다.
- 소주제는 사용자가 실제로 겪는 상황 중심으로 만든다.
- 모든 과정 마지막에는 기존 마일고사형 `마스터 인증`을 연결한다.

최종 구조:

```text
대주제: 호텔 / 항공 / 카드·포인트 / 상품권
중주제: 브랜드 또는 과정
소주제: 듀오링고식 레슨 노드
마지막: 마스터 인증 = 기존 마일고사
```
